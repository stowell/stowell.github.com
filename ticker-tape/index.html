Opposite every rant there's an anti-rant awaiting extraction

Amen to Ryan Dahl, who among us can't get behind simplicity and in particular simplicity that we once had in our hands in the seventies but let slip away: 'In the past year I think I have finally come to understand the ideals of Unix: file descriptors and processes orchestrated with C. It's a beautiful idea.'

The title of the piece is I hate almost all software which is fine, I get it, most every simple thing is too fraught and our field tends to select for people who have the patience enough to outlast the minutaue of getting an actual program to run on real computers most of the time. But everything is awful and the man himself has spent a good chunk of his time writing software. Let's see if we can't turn this anger inside out:

Those of you who still find it enjoyable to learn the details of, say, a programming language - being able to happily recite off if NaN equals or does not equal null - you just don't yet understand how utterly fucked the whole thing is. If you think it would be cute to align all of the equals signs in your code, if you spend time configuring your window manager or editor, if put unicode check marks in your test runner, if you add unnecessary hierarchies in your code directories, if you are doing anything beyond just solving the problem...

becomes (in my code-on-your-phone agenda)

  • Make equality obvious.
  • Allow code to come from outside an editor.
  • Avoid configuration, avoid the invisible.
  • Pick a character set and keep it consistent.
  • Let code live outside a file system.

How else does anything happen other than the continual shameless stealing of ideas

There's a lot in Eve, both in spirit and implementation, that's worth stealing or at least spreading as far and wide as possible so that it's not lost: the program can never blow up, only produce error records, everything is data manipulation, the program and the database are the same thing and move through time together, code is a part of prose, code executes when there's something to execute on.

An off-hand Chris Granger tweet mentioned that Slack messages are valid Eve programs and I wonder just how far they've taken that. The fact that there are braces and pound signs in the language makes me think that no one's written large programs in Eve on their phones. That's exactly where I want to go with these ideas and if we have to toss some punctuation to get there well.

When my mind starts going down this path I worry that I'm stuck in the same rut I've been stuck in since I was nineteen, wanting to write a programming language because the people who had written programming languages seemed super-smart. Nowadays I'm not so scared of ruts. Just last week we were talking about making a track-to-track recommender for a prototype and it's the same story for the thirty-first time: pull back a set of plausible recommendations, take the subset that actually makes sense together, return it in some order. The rules around ordering change but are always similar and obvious to the listener when you get them wrong. The younger me would say we've written this program too many times but now I'd day we've just got to get better at writing it.
